Problema & Solução

Fluxo Principal

Como funciona a cobrança

Da importação do PDF ao registro do resultado — 5 etapas totalmente automatizadas.

1
Upload de PDF
Usuário faz upload de até 20 boletos no dashboard Angular. Backend extrai valor e vencimento via leitura de texto do PDF.
2
Persistência no banco
Recebível salvo no PostgreSQL com status pending. Upload duplicado é idempotente via hash SHA-256.
3
Evento Kafka: receivable.created + charging.requested
Backend publica dois eventos: um de criação do recebível e um solicitando cobrança, com contatos da empresa incluídos.
4
n8n processa e notifica
n8n consome o evento do Kafka, escolhe o canal (WhatsApp via Evolution API, e-mail etc.) e realiza a cobrança operacional.
5
Resultado registrado
n8n publica charging.succeeded ou charging.failed. Backend consome, atualiza o status do recebível para charged e persiste o resultado.

RLS habilitado + políticas deny-all no Supabase. NestJS é o único boundary de acesso. JWT validado a cada request.

Segurança

Papéis e controle de acesso

Cada usuário pertence a uma organização com um role. Acesso a empresas é granular e controlado pelo admin.

👑 Admin
✓ Cria e gerencia usuários da organização
✓ Acessa todas as empresas (read_write implícito)
✓ Concede/revoga acesso de usuários a empresas
✓ Importa boletos e solicita cobranças
✓ Aciona recobrança manual (1–20 recebíveis)
✓ Bootstrap inicial do sistema
👤 User
✓ Acessa apenas empresas com acesso explícito
✓ Importa boletos (se tiver read_write)
✓ Visualiza recebíveis das suas empresas
✗ Não cria outros usuários
✗ Não vê empresas sem acesso configurado
✗ Auto-cadastro não existe — só via admin
🔒 Níveis de acesso por empresa
Cada user pode ter read (visualização) ou read_write (importação + cobrança) por empresa individualmente.
Roadmap

Limitações conhecidas & próximos passos

⚠️ Limitações atuais

PDFs escaneados
Sem OCR ainda — boletos de imagem não são extraídos. Apenas PDFs com texto embutido.
Kafka não transacional
Recebível é salvo e depois o evento é publicado. Falha no Kafka deixa o recebível sem cobrança disparada.
Arquivo PDF não armazenado
invoice_file_path fica null — integração com storage ainda não implementada.

🚀 Próximos passos

Transactional Outbox
Garantir que evento Kafka seja publicado mesmo após falha, usando padrão outbox no banco.
OCR para boletos escaneados
Integrar extração por imagem para suportar boletos digitalizados.
Storage de PDFs
Integrar Supabase Storage ou S3 para persistir os arquivos originais dos boletos.
